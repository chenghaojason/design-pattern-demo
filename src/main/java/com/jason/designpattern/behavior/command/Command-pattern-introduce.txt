命令模式（Command Pattern）：
定义：将一个请求封装为一个对象，从而是我们可用不同的请求对客户进行参数化，对请求排队或者记录请求日志，以及支持可撤销的操作。

命令模式的四个角色：
    1.抽象命令类 Command：抽象命令一般是个接口，声明了用于执行请求的方法等。
    2.具体命令 ConcreteCommand：抽象命令的子类，实现了抽象命令中的方法，它对应具体的接收者对象，绑定接收者对象的动作。
    3.调用者 Invoker：即请求发送者，也成为请求者，它通过命令对象来执行请求。
    4.接收者 Receiver：接收者执行与请求相关的操作，它具体实现对请求的业务处理。
